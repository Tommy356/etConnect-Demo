Overview

ETConnectServer provides a functionality similiar to NSPortNameServer. However, as the NSPortNameServer class is not available on iOS and etConnect relies on NSStream classes rather than ports, implementation and API differes in several parts from what NSPortNameServer has been designed for.

There’s a single shared in stance of ETConnectServer for every app – no need to create more than one instance.

startWithOptions:

Starts the server with the options supplied.

- (BOOL)startWithOptions:(ETServerOptions)options

Parameters

options:

a combination of one or more ETServerOptions values.

Return Value

TRUE, if the server is starting, FALSE if an error occured.

Discussion

The following options may be supplied, when starting the server:

  • ETServerOptionSearch: The server will also start to discover other hosts on the network
  • ETServerOptionPublish: The server will publish the general ETConnect service
  • ETServerOptionPublishService: The server will publish its management service (requires ETServerOptionPublish)
  • ETServerOptionPeerToPeer: Enables bluetooth communication (IOS7 only)
