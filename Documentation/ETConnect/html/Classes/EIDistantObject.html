Overview

A Distant Object is a placeholder for a Proxy provided by a remote host. Typically, you won’t create instances of this class directly, but use the routines provided by the EIConnection class to either publish local objects or connect to remote objects published by other hosts.

Properties

platformType

Platform encoding: returns a combination of the values defined above The platform types are used to differentiate between the coding of values (i.e. NSInteger/NSUInteger) which have different sizes on 32/64 bit platforms. The following definitions are available:

@property (nonatomic, readonly) ETServerPlatformType platformType

Discussion

  • ETServerPlatformType_16Bit = ( 1 << 0),
  • ETServerPlatformType_32Bit = ( 1 << 1),
  • ETServerPlatformType_64Bit = ( 1 << 2),
  • ETServerPlatformType_iOS = ( 1 << 4),
  • ETServerPlatformType_OSX = ( 1 << 5),

Remote calls to different platform types will usaually work withoput a protocol, if only objects are passed as parameters. Structure- and NSInteger/NSUInteger parameters are refused without a defined protocol.

proxyWithTarget:connection:

Creates and returns a proxy for a target object.

+ (EIDistantObject *)proxyWithTarget:(unsigned)anObject connection:(EIConnection *)aConnection

Discussion

This routine is used internally, to create proxies for objects returned upon remote requests.

Departure from the OpenStep/MacOS spec - the type of a target is an integer, not an id, since we can't safely pass id's
between address spaces on machines with different pointer sizes.

setProtocolForProxy:

Sets a protocol for a proxy retrieved from a remote host.

- (void)setProtocolForProxy:(Protocol *)aProtocol

Discussion

The usage of protocols is highly recommended to speed up network communication and to enable safe coding of data structures and NSInteger/NSUInteger parameters between 32-bit and 64-bit platforms. NOTE, that cross-platform calls using such parameters will require a protocol to be set; otherwise an exception will be raised.

For a more intense dicussion on this topic please refer to the etConnect framework documentation.
